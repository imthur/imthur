## Hey. I'm Arthur.

I like building systems that have real rules behind them.

I do not get that excited about projects that are only pretty screens with fake data. I enjoy the part where the application has to deal with flows, permissions, validation, state, databases, APIs, deployment, production-ish bugs — the kind of stuff that pushes code out of tutorial mode and into actual software.

My main focus right now is **back-end development with Python, Django and Django REST Framework**, but I also like closing the loop when a project needs front-end, integration work, automation or a more careful visual layer.

My background in **data analysis** influences the way I think about software: modeling, consistency, information flow, structure and decisions that will not make everything painful two weeks later.

---

## What I am working on now

- building APIs and web systems with business rules closer to real-world applications;
- improving architecture, permissions, authentication, validation and testing;
- getting better at deployment, CI, logs, observability and operational routines;
- growing as a back-end developer without ignoring product thinking, UI and user experience.
